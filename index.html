SAP-1 SIMULATOR — ABOUT
WHAT IS SAP-1?

SAP stands for Simple As Possible. The SAP-1 is an 8-bit educational CPU architecture designed by Albert Paul Malvino and Jerald A. Brown in their textbook Digital Computer Electronics (1977). It is not a commercial chip — it was purpose-built to teach how a CPU works at the gate level, starting from nothing.

INSPIRED BY BEN EATER

Ben Eater built a physical SAP-1 on breadboards using 74HC-series logic chips and documented the entire build on YouTube. His series became one of the most accessible explanations of computer architecture ever made — showing exactly how flip-flops, adders, decoders, and buses combine into a working CPU you can hold in your hand.

WHAT DOES THIS SIMULATOR DO?

This simulator replicates the SAP-1 in software. It executes assembly code cycle-by-cycle, showing exactly what happens inside the CPU at every T-state: which registers change, which control signals fire, and what data is on the bus.

  • 8-bit data path, 4-bit addresses, 16 bytes of RAM
  • 6 T-states per instruction (FETCH + EXECUTE)
  • 9-instruction ISA: LDA, ADD, SUB, STA, LDI, JMP, JC, JZ, OUT, HLT
  • Zero and Carry flags for conditional branching
HOW TO USE
  • Write assembly in the editor, or pick an example from the dropdown
  • Click ASSEMBLE & LOAD to compile into RAM
  • Use STEP to advance one clock cycle at a time
  • Use RUN to let it execute automatically
  • Adjust speed with the slider (0.5 Hz → 64 Hz)
  • Click any RAM cell to edit its value directly
  • Comments after ; are ignored by the assembler
INSTRUCTION SET

ADD nA ← A + RAM[n] (sets Zero, Carry flags)
SUB nA ← A − RAM[n] (sets Zero, Carry flags)
STA nRAM[n] ← A
LDI nA ← n (immediate, 4-bit value 0–15)
JMP nPC ← n (unconditional jump)
